# 技术栈说明

本文档用于快速熟悉本仓库所用技术与主要代码位置；内容以当前仓库 `pom.xml` 与 `src/main/java` 为准，若版本升级请以构建文件核对。

## 读者目标

- 了解依赖版本、Spring AI 接入方式与 HTTP 入口。
- 理解“用户消息 -> ChatClient -> 可选 Tool 调用 -> 文本回复”的最小链路。
- 用“面试要点”小节准备对外表述，避免夸大项目范围。

## 仓库与源码导航

| 路径 | 说明 |
|------|------|
| `pom.xml` | Maven 坐标、Java/Spring Boot/Spring AI 版本与依赖 |
| `src/main/java/com/springailab/lab/SpringAiLabApplication.java` | Spring Boot 入口 |

## 配置说明

### 大模型 profile

- `deepseek`：读取 `application-deepseek.yml`
- `qwen`：读取 `application-qwen.yml`

默认由 `spring.profiles.active` 决定，也可通过环境变量覆盖：

- `SPRING_PROFILES_ACTIVE=qwen,mysql,redis-vector`

### 外部调用配置（Tool 用）

- 前缀：`lab.external.post.endpoints.<profile>`
- 关键 profile：
  - `weather`
  - `demo-echo`
  - `user-query`
  - `vector-search`

---

## 模块与调用关系

1. `ChatController` 接收用户请求
2. `ChatClient` 调用模型并注册 `WeatherTools`
3. 模型决定是否触发 `@Tool`
4. Tool 通过 `ExternalJsonPostClient` 发起 HTTP POST JSON
5. 返回摘要结果给模型，最终输出 `reply`

---

## 面试要点（简版）

- 本项目采用方案 A：`@Tool` + function calling 的最小闭环。
- 业务查询与向量检索通过项目内 API 分层，避免 Tool 直接耦合底层细节。
- 生产化重点是可控性（权限/校验）、可观测（traceId）、稳定性（重试/熔断）。
