## 3. 代码改动总览

### 3.1 对话入口与编排层

#### 变更点

- `ChatController` 从“直接调用 `ChatClient`”改为“委托 `ChatOrchestrator`”。
- 保留同步接口：`POST /chat`。
- 新增流式接口：`POST /chat/stream`（`text/event-stream`）。
- 请求体扩展为：`message + conversationId`。

#### 涉及文件

- `src/main/java/com/springailab/lab/web/ChatController.java`
- `src/main/java/com/springailab/lab/domain/chat/service/ChatOrchestrator.java`

### 3.2 会话记忆（Redis）

#### 变更点

- 新增会话存储抽象与 Redis 实现。
- 按 `conversationId` 管理会话消息。
- 写入时执行保留策略：`maxMessages` 裁剪 + `ttlSeconds` 过期。
- 历史消息参与 prompt 构建（`U:` / `A:` 前缀区分用户与助手消息）。

#### 涉及文件

- `src/main/java/com/springailab/lab/domain/chat/service/ConversationMessageStore.java`
- `src/main/java/com/springailab/lab/domain/chat/service/RedisConversationMessageStore.java`
- `src/main/java/com/springailab/lab/domain/chat/config/ChatSessionProperties.java`

### 3.3 SSE 事件协议

#### 变更点

- 事件类型：`token`、`done`、`error`。
- 统一事件载荷对象：`conversationId + content`。
- 处理了正常完成、异常终止、超时与客户端断开资源释放。

#### 涉及文件

- `src/main/java/com/springailab/lab/domain/chat/service/ChatEventPayload.java`
- `src/main/java/com/springailab/lab/domain/chat/service/ChatOrchestrator.java`

### 3.4 可观测与估算成本

#### 变更点

- 新增基础指标：
  - `chat_stream_requests_total`
  - `chat_stream_errors_total`
  - `chat_stream_latency`
- 新增估算成本指标：
  - `ai_estimated_cost_total`
- 成本计算来源：`usage.promptTokens + usage.completionTokens`，按“每千 token 单价”折算。
- 流式路径按请求级收集，避免按 chunk 重复累计。

#### 涉及文件

- `src/main/java/com/springailab/lab/domain/chat/service/ChatOrchestrator.java`
- `src/main/java/com/springailab/lab/domain/chat/config/ChatCostProperties.java`

### 3.5 演示页

#### 变更点

- 新增最小前端演示页（无框架）：
  - 输入 `message` + `conversationId`
  - 调用 `/chat/stream`
  - 展示流式返回和状态变化

#### 涉及文件

- `src/main/resources/static/index.html`
